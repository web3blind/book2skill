# Book Type Patterns

Use these patterns to choose the shape of the generated practical skill.

## Writing / editing / communication

Best shape: editor + writing coach.

Useful modes:
- draft from brief
- rewrite unclear text
- shorten without losing meaning
- critique text
- preserve voice while improving clarity
- adapt to channel/audience

References:
- editing checklist
- anti-patterns
- before/after examples as abstract patterns
- tone and voice rules

## Programming / system design / technical architecture

Best shape: technical advisor + decision guide + reviewer.

Useful modes:
- choose approach
- review design
- debug by framework
- generate implementation checklist
- compare trade-offs
- explain concept using book model

References:
- frameworks
- decision tables
- implementation checklists
- pitfalls
- chapter/topic index

## Product / management / operations

Best shape: diagnostic coach + planning facilitator.

Useful modes:
- diagnose situation
- choose next action
- run a retrospective
- plan operating cadence
- review a proposal
- identify risks and anti-patterns

References:
- diagnostics
- meeting/workshop templates
- principles
- operating checklists

## Marketing / sales / positioning

Best shape: campaign strategist + critique assistant.

Useful modes:
- define positioning
- critique landing/copy
- plan campaign
- segment audience
- improve offer
- generate test hypotheses

References:
- frameworks
- message checklist
- examples/patterns
- objection handling

## Self-development / psychology / negotiation

Best shape: coach + scenario simulator + reflection guide.

Useful modes:
- prepare for situation
- debrief event
- choose response
- roleplay
- identify cognitive trap
- make practice plan

References:
- mental models
- prompts/questions
- anti-patterns
- exercises

## Reference / handbook / encyclopedia

Best shape: navigator + lookup + concise explainer.

Useful modes:
- lookup concept
- compare terms
- find relevant chapter/section
- make quick checklist
- apply one concept to a case

References:
- glossary
- topic index
- decision tables
- chapter index

## Mixed books

If the book mixes theory and practice, choose one primary operating model and add references for the rest.
Do not make the skill try to be everything.
